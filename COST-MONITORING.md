# 费用监控指南 - Vercel 免费套餐

## 📊 Vercel 免费套餐限制（2024年）

### 免费套餐包含：

| 资源类型 | 免费额度 | 说明 |
|---------|---------|------|
| **带宽** | 100 GB/月 | 所有传输的数据总量 |
| **函数调用** | 100 GB-小时/月 | Serverless 函数执行时间 |
| **构建时间** | 6000 分钟/月 | 项目构建时间 |
| **部署数量** | 无限制 | 可以无限次部署 |
| **项目数量** | 无限制 | 可以创建多个项目 |
| **域名** | 无限制 | 可以绑定自定义域名 |

### ⚠️ 什么会产生费用？

只有在**超出免费额度**时才会产生费用：

1. **带宽超过 100 GB/月**
   - 每个页面访问都会消耗带宽
   - 图片、视频等大文件会消耗更多带宽
   
2. **函数执行时间超过 100 GB-小时/月**
   - 你的项目主要是静态页面，函数使用很少
   - 不太可能超出此限制

3. **构建时间超过 6000 分钟/月**
   - 每次部署都会触发构建
   - 一般每次构建 1-3 分钟

- ✅ **主要是静态内容**：首页、项目页面都是静态的
- ✅ **轻量级资源**：没有大量图片或视频

### 估算访问量：

假设平均每个页面大小为 **500 KB**（包括 HTML、CSS、JS）

```
100 GB = 100,000 MB = 100,000,000 KB

每月可支持访问次数 = 100,000,000 KB ÷ 500 KB = 200,000 次访问
每天可支持访问次数 = 200,000 ÷ 30 = 约 6,666 次访问
```

**结论**：对于个人项目或小型网站，免费套餐绰绰有余！

## 📈 如何监控使用量

### 1. Vercel 控制台监控

访问 Vercel 控制台查看实时使用情况：

1. 登录 [Vercel Dashboard](https://vercel.com/dashboard)
2. 点击你的项目
3. 进入 **Settings** → **Usage**
4. 查看：
   - 当前月份的带宽使用量
   - 函数执行时间
   - 构建时间

### 2. 设置使用量警报

Vercel 会在你接近限制时自动发送邮件警告：
- 达到 80% 时会收到第一次警告
- 达到 100% 时会收到最终警告

### 3. 查看分析数据

在 Vercel 控制台的 **Analytics** 标签中可以看到：
- 页面访问量
- 访问来源
- 性能指标

## 🛡️ 如何避免意外费用

### 策略 1: 优化资源大小

```bash
# 构建生产版本（已优化）
npm run build

# 检查构建产物大小
```

Next.js 会自动：
- 压缩 JavaScript 和 CSS
- 优化图片
- 代码分割（Code Splitting）

### 策略 2: 使用 CDN 托管大文件

如果需要使用大量图片或视频：
- 使用免费图床服务（如 Cloudinary、ImgBB）
- 使用 GitHub 托管静态资源
- 使用专门的视频托管服务

### 策略 3: 缓存策略

你的项目已经在使用缓存：
```typescript
// users/page.tsx
{cache: "no-store"}  // 这会每次都请求新数据
```

**建议**：对于不经常变化的数据，使用缓存：
```typescript
{
  next: { revalidate: 3600 }  // 每小时更新一次
}
```

### 策略 4: 限制构建频率

- 不要频繁部署（每次部署都会消耗构建时间）
- 在本地充分测试后再部署
- 使用 `git push` 触发自动部署，而不是手动部署

## 🚨 紧急情况处理

### 如果收到超额警告：

1. **立即检查使用情况**
   - 登录 Vercel 控制台
   - 查看哪个资源超额

2. **临时措施**
   - 暂停项目（在设置中）
   - 等待下个月重置

3. **长期解决方案**
   - 优化资源大小
   - 迁移大文件到 CDN
   - 考虑升级到付费计划（如果项目有价值）
